Vad är GitHub Pages?

GitHub Pages är ett gratis sätt att visa en webbplats på internet direkt från ett GitHub-repo. Du behöver inte hyra en egen server för enkla webbsidor.

Tänk så här

GitHub Pages är som en digital anslagstavla. Du lägger upp färdiga webbfiler, till exempel HTML, CSS, JavaScript och bilder. GitHub visar sedan filerna som en riktig webbsida.

GitHub Pages passar bra för

  • Portfolios
  • Enkla webbsidor
  • Dokumentation
  • Små HTML/CSS/JS-spel
  • Demos och prototyper
  • Statiska exports från t.ex. Vite

GitHub Pages passar sämre för

  • Appar med egen backend
  • Databaser med serverlogik
  • Hemliga API-nycklar
  • Inloggning som kräver säker serverkod
  • Python, Node, PHP eller C# på servern
  • Lokala program som ffmpeg, Piper eller Ollama
Kort sagt: GitHub Pages visar färdiga webbsidor. GitHub Pages kör inte en egen backend åt dig.

Snabbregel

✅ Bara färdiga webbfilerHTML, CSS, JavaScript och bilder → GitHub Pages kan passa
⚠️ Behöver server eller säker dataBackend, databas, hemliga nycklar eller lokala program → Välj troligen en annan lösning

Ordlista

Statisk webbplats En webbplats som bara består av färdiga filer (HTML, CSS, JS, bilder) och inte behöver en server som bearbetar data.

En statisk webbplats är en sida som redan är färdig när den skickas till webbläsaren. Den består oftast av filer som HTML, CSS, JavaScript och bilder. Webbläsaren läser filerna och visar sidan direkt.

Det betyder att sidan inte behöver en egen server som räknar ut saker i bakgrunden varje gång någon besöker den. Därför passar statiska webbplatser ofta bra på GitHub Pages.

Exempel: en portfolio, en informationssida, en enkel dokumentationssida eller ett litet spel byggt med HTML, CSS och JavaScript.

Backend Den del av ett system som körs på en server, till exempel kod i Node.js, Python eller PHP som hanterar logik och data.

Backend är den del av en app som körs på en server, alltså inte direkt i besökarens webbläsare. Den kan till exempel hantera inloggning, spara data, prata med en databas eller skydda hemliga nycklar.

GitHub Pages kan visa färdiga webbsidor, men det kan inte köra en egen backend åt dig. Om ditt projekt behöver backend behöver du oftast en annan lösning, till exempel Vercel, Render, Railway, Supabase eller en egen server.

Databas Ett system för att spara, hämta och hantera data, till exempel MySQL, PostgreSQL, Supabase eller Firebase.

En databas är en plats där en app kan spara och hämta information. Det kan vara användare, kommentarer, bilder, produkter, poäng, bokningar eller annan data.

GitHub Pages har ingen egen databas. En statisk sida kan ibland läsa data från en extern tjänst, men om appen behöver spara data säkert eller hantera användardata behöver du oftast en databaslösning som Supabase, Firebase eller en egen backend.

API-nyckel En hemlig kod som ger tillgång till en tjänst. Hemliga nycklar ska aldrig vara synliga i webbläsaren.

En API-nyckel är som ett lösenord som ger tillgång till en tjänst. Den kan till exempel användas för att prata med en AI-tjänst, en betaltjänst, en karttjänst eller en databas.

Om en API-nyckel är hemlig ska den aldrig ligga öppet i kod som körs i webbläsaren. På GitHub Pages kan användaren i princip se frontend-koden. Därför behöver hemliga nycklar oftast hanteras via en backend eller serverlösning.

Build Att omvandla källkod till färdiga filer som kan visas i en webbläsare, ofta till en mapp som heter dist eller build.

Build betyder att projektets källkod görs om till färdiga filer som webbläsaren kan använda. Verktyg som Vite, React, Vue eller Svelte kan till exempel skapa en mapp som heter dist eller build.

GitHub Pages kan ofta visa resultatet av en build, men det kör inte själva appservern. Om din build skapar vanliga HTML-, CSS- och JavaScript-filer kan projektet ofta fungera på GitHub Pages.

Deploy Att publicera en webbplats så att den blir tillgänglig på internet, till exempel via GitHub Pages eller Netlify.

Deploy betyder att du publicerar ditt projekt så att andra kan besöka det på internet. När du deployar flyttas eller kopplas dina filer till en plats där webbläsare kan nå dem.

GitHub Pages är ett sätt att deploya statiska webbsidor. Andra vanliga alternativ är Netlify, Vercel och Cloudflare Pages. Vilken lösning som passar beror på om projektet bara behöver visa filer eller om det också behöver backend.

Lokal körning Att köra ett program på sin egen dator istället för på en server på internet.

Lokal körning betyder att projektet körs på din egen dator i stället för på internet. Det kan till exempel vara en app som använder Python, ffmpeg, Piper, Ollama, Stable Diffusion eller andra program som finns installerade lokalt.

GitHub Pages kan inte köra sådana lokala program åt besökaren. Om projektet kräver lokala program behöver det oftast köras på din dator, på en egen server eller byggas om till en annan lösning.
